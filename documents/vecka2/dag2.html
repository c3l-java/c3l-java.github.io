## **Statiskt innehåll i Play.** * Assets * Lägg vad du vill här * Default-mapparna är inte “låsta” utan kan bytas ut * = Enkel re-routing av “döda” filer * Ingen hårdkodning av absolut URL * Bra att inte ha ett “magiskt” mappnamn som t.ex. public, som aldrig kan ändras * Din mapp _kan_ flyttas, om det skulle bli aktuellt för din applikation * (Testa ändra till “filer”) * Det sker s.k. reverse routing för att hitta din fil * Din fil ligger på: * public/main.css * I din view: * @routes.Assets.at("test.css") * I din routes: * /assets → Assets.at(…) * LESS * [lesscss.org](http://lesscss.org) * CSS + extra features * För att få syntax highlighting: * Eclipse → Preferences, sök efter Content Types * Välj Text → CSS * Välj Add → skriv in *.less * Testa att lägga in main.less med content: * body{ background-color: lighten(green, 10%) } * Det fungerar inte! * LESS är ingenting som webbläsare förstår * Måste “preprocessas” * Detta tar Play hand om * _Managed_ assets * Skapa “mappen” (som ser ut som ett paket) assets inuti app * Mappas mot public * Testa med en txt-fil vilken som kommer först * Nu får vi tillgång till några häftiga funktioner * (Du kan säkert göra dina egna moduler som gör grejer med dessa filer) * LESS * (Lägg in main.less och länka till main.css) * LESS-minifering * lägg till .min.css istället * Javascript-minifiering * lägg till min.js istället * Allt detta sker automatiskt endast om du har watch-mode aktiverat (~) * Lika bra att stänga av om det är en live-server * Genererade filer hittar du under target/scala…/resource_managed/main/public/ ### **Laboration: Få LESS-filer att kompilera** *   ### **Uppgift: Skapa tomma controllers/enkla views/routes för varje sida som kommer behövas i din Webbshop** * Exempel: 1) Startsida 2) Kategorisida 3) Produktsida 4) Visa Shoppingvagn 5) Checkout * Tex Start → start.scala.html * Category → * category.showOne.scala.html * category.listAll.scala.html * category.edit.scala.html * … Detta är bara exempel … * Kom på alla olika views som kan tänkas behövas, men fyll dem med **väldigt enkelt innehåll** * Var beredda på att vi ska lyfta över mycket av er webbshops-kod från era gamla projekt!
